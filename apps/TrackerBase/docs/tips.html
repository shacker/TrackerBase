Tip Server

This is the script that is currently driving The BeOS Tip Server. Also included are 40 random tip files in the data folder so you can generate a mini-version of the site. Structurally, it isn't much different from the other scripts -- an index page with sub-pages. However, this index page groups links by category, rather than just running them down. It also creates a "special" group of links at the top of the page for recent entries -- these are flagged by a "New" attribute that carries a yes or no value. Toggle that attribute to "n" from Tracker, and it won't show up in the recent list next time the site is generated.

This is also the only script that uses queries to gather data, rather than straight directory crawls. Queries allow for much more interesting results, but have some gotchas. You must, for instance, take care to filter out any TrackerBase files that are in the Trash. You also have to make sure to specify the project name in the "project" attribute so your queries don't accidentally turn up TrackerBase files from other projects.

You'll notice that the top of the script has a long list of if/then statements relating to banner ads. If the AdKey attribute equals X, then the associated graphic and URL are... . AdGraphic and AdURL were initially attributes of their own, but Tracker has no mechanism for linking one attribute to another, as you might do in a commercial database. Linking them this way simplifies administration and prevents foul-ups.

As with the news site, URLs and e-mail addresses are only created on tip pages when those attributes are not blank.
