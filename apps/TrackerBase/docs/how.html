How TrackerBase Works

Every folder on your system can potentially be used as a database, whether or not it contains TrackerBase files. TrackerBase scripts use a combination of techniques to construct Web sites from folders full of files. In all cases, the basic process is this:

  1. Establish some regular directories for input, output, and "parts," or chunks of static HTML to be integrated into the pages you build.

  2. Break an existing Web page into consecutive pieces, consisting of things that stay the same and things that change. Create multiple small files out of the static items. Very small chunks of HTML, such as a single <P> tags, can simply be dropped into scripts with the echo command.

  3. Define elements that are different for every page as variables, and create BeOS attributes for them, if they don't exist already.

  4. If you have "main" bodies of content, as most Web pages do, store that in the TrackerBase text file itself.

  5. Create a list of files to be turned into Web pages. This list can exist only in memory, or it can be written to a text file for temporary storage. It can be a list of every file in a directory, or of files that match some criteria, such as a BeOS query result. Each line in the list represents one of your files (records). Go through this list one line at a time and hoover the attributes you need out of the corresponding file. In most cases, assign the contents of each attribute to a variable. See Attributes for more on storing, managing, and extracting attributes).

  6. In some cases, scripts will run extra tests and perform extra actions when certain conditions are true. For example, the tipserver and news_site scripts will create linked URLs and e-mail addresses if and only if URLs or e-mail addresses are present.

  7. In a specific order, concatenate static HTML chunks from the parts directory with variables in memory. Write the results to an HTML file and go on to the next one.

  8. Build index (linking) pages similarly. Some of these scripts build the index page and the linked pages simultaneously, branching off every time a link is created to construct the page that's being linked to.
