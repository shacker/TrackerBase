TrackerBase II : Dynamic site generation techniques

Most of the TrackerBase system in this package is designed for creating static sites from the BFS database - for generating normal HTML pages that can be uploaded to any web server.

"Real" database-driven web sites assemble pages on the fly, as they're requested. This portion of the TrackerBase system is used for dynamic web site generation from the BFS database, for use with any cgi-capable BeOS web server, such as Robin Hood or Apache.

Getting this part of TrackerBase up and running is not as easy as the rest of the TrackerBase system. You will probably need at least basic experience working with perl, and you will need to tweak a lot of variables in the sample files provided here. Documentation on these files is not nearly as complete as on the rest of the TrackerBase system. I probably will not be able to support you if you run into problems, sorry.

In order for this part of the TrackerBase system to work, you must have installed the basic TrackerBase package, perl for BeOS, as well as the BFS attribute extraction modules for perl. For instructions, see:

Installing and using CPAN perl modules

You must also have a BeOS web server up and running.

Open /boot/apps/TrackerBase/TrackerBase II and you'll find four files. You'll need to put these into place manually, as follows.

There are two components to the dynamic TrackerBase II page generation scheme:

1) The script and associated files that regenerate the site's menu pages, and
2) The script and associated files that generate individual web pages on the fly.

On betips.net, step 1 (menu page generation) is accomplished by running TrackerBase.pl from the command line. TrackerBase.pl invokes TrackerBase.pm as it runs. Therefore:

Place TrackerBase.pl in ~/config/bin
Place TrackerBase.pm in ~/config/lib/perl5/site_perl/5.005

On betips.net, step 2 (live page generation from attributes) is accomplished by invoking chunga.pl* from a web browser, which relies in part on HTML stored in chunga.tip.template. Therefore:

Place chunga.pl in your site's cgi-bin directory
Place chunga.tip.template in a "templates" folder on your site.

You will probably have to tweak variables in all of these files to match locations on your machine.

*This version of chunga.pl was written mostly by BeGirl (Tori) - thanks for your contribution, Tori.
